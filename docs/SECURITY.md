# Security Release Process

Security vulnerabilities or suspected security vulnerabilities should be reported to the maintainers privately so issues can be investigated and fixed before wide disclosure.

## Reporting a Vulnerability

Do not file public issues for security vulnerabilities.

Please contact the maintainers privately with:

- a descriptive summary
- reproduction steps or proof of concept
- affected versions and configuration details
- any known mitigations or related dependency information

Feedback will be sent as soon as practical with the next investigation steps.

## Disclosure

Once a vulnerability is confirmed:

1. maintainers assess severity and impact
2. a fix or mitigation plan is prepared
3. supported releases are patched as appropriate
4. public disclosure follows once a fix is available or mitigations are clear
